Combining sequencing depth and SNV ratio AnnData objects¶

GEI-NJ uses sequencing depth and SNV ratio AnnData objects as input. The depth object, ad_depth, contains the corresponding sequencing depths, while the ratio object, ad_ratio, contains SNV ratios. These two objects are combined into a single AnnData object adata using comb_depth_ratio.

In [2]:
## Load in SNV ratio and depth
ad_depth = ad.read_h5ad("unit_depth.h5ad")
ad_ratio = ad.read_h5ad("unit_ratio.h5ad")

## Combine depth and ratio into a single AnnData object
adata = comb_depth_ratio(ad_depth, ad_ratio)

## Set group as the row names of the combined object
adata.obs["group"] = adata.obs_names.astype(str)

SNV filtering¶

SNV filtering can be performed based on sequencing depth and mutation ratio using filt_snv_by_depth_ratio. In this step, min_tot_depth specifies the minimum total sequencing depth required for each retained SNV, min_mut_depth specifies the minimum mutant UMI support, and min_mut_avg_ratio specifies the minimum average mutation ratio. The average mutation ratio is computed within this function as mutated UMI divided by total UMI, which is equivalent to a depth-weighted average of unit-level mutation ratios.

In [4]:
## Filter SNVs based on depth and ratio
adata = filt_snv_by_depth_ratio(adata, min_tot_depth=20, min_mut_depth=5, min_mut_avg_ratio=0.1)

Construction of callable layers¶

Callable layers can be generated using add_callable_layer. This step records whether each SNV is callable in each cell group based on sequencing depth. The minimum depth required for a callable entry can be specified using min_callable_depth.

In [5]:
## Construct callable layer based on depth
adata = add_callable_layer(adata, min_callable_depth=3)

Revision of SNV ratios¶

To harmonize mutation signals across loci and reduce sensitivity to continuous ratio noise caused by cell aggregation and gene expression bias, SNV ratios are discretized by default using revise_snv_ratio. By default, ratios lower than 0.1 are set to 0, ratios higher than 0.9 are set to 1, and intermediate ratios are assigned to 0.5.

In [7]:
## By default, the original SNV ratio will be moved to the adata.layers["original_ratio"] layer
adata = revise_snv_ratio(adata)

Imputation of non-callable SNV ratios¶

GEI-NJ supports imputation of non-callable SNV entries using impute_non_callable_by_snv_mean. This function calculates the per-SNV mean from callable revised SNV ratios and uses this value to impute non-callable entries. The imputed SNV ratio matrix is stored in adata.layers["geinj_state_imputed"] and adata.X by default.

In [8]:
## By default, this step stores the imputed SNV ratio matrix in adata.layers["geinj_state_imputed"]
## and updates adata.X to the imputed matrix
adata = impute_non_callable_by_snv_mean(adata, callable_layer="callable", output_layer="geinj_state_imputed")

Elbow detection (recommended)¶

GEI-NJ does not require a fixed strategy for selecting the number of PCs used for NJ tree construction. Users can specify this number based on scree plots, elbow points, prior knowledge, or other user-defined criteria. Here, we use the elbow point of the scree plot to guide PC selection. The elbow point is detected outside the core GEI-NJ functions using the KneeLocator function from the Python package kneed.

In [9]:
from kneed import KneeLocator

## Detect the elbow point with the additional KneeLocator function
n_comps = min(adata.n_obs-1, adata.n_vars, 50)
sc.pp.pca(adata, n_comps=n_comps)
explained_var = adata.uns['pca']['variance_ratio']
explained_var_cum = explained_var.cumsum()

# Detect elbow point
kl = KneeLocator(range(1, len(explained_var)+1), explained_var, curve="convex", direction="decreasing")
elbow_PC = kl.elbow

plt.figure(figsize=(5,4))
plt.plot(range(1, len(explained_var)+1), explained_var, 'o-', markersize=3, label="Individual explained variance")
plt.plot(range(1, len(explained_var)+1), explained_var_cum, 's-', markersize=3, label="Cumulative explained variance")
plt.axvline(elbow_PC, color='r', linestyle='--', label=f'Elbow at PC {elbow_PC}')
plt.xlabel('Principal Component')
plt.ylabel('Explained Variance Ratio')
plt.legend()
plt.title('Scree plot of PCA')
plt.show()
plt.close()

GEI-NJ trees are generated from the SNV PC matrix using build_and_plot_nj(). In the examples below, K_tree=2 indicates that PC 1 and PC 2 are used for tree construction. GEI-NJ adds a non-SNV outgroup as a pseudo-founder cell group to root the inferred developmental structures. Several example layouts of build_and_plot_nj() are shown below.

LOO assessment¶

To assess the stability of GEI-NJ tree topology with respect to PC selection, we demonstrate the leave-one-out (LOO) option implemented in build_and_plot_nj(). In the following example, LOO assessment was performed using the first 4 PCs by setting pc_loo=True and K_pc_loo=4. GEI-NJ then generated replicate NJ trees by excluding one of these PCs at a time and calculated LOO support for each internal split in the reference tree as the percentage of LOO trees in which that split was recovered. For visualization, only LOO support values greater than or equal to pc_loo_min_support are displayed on the GEI-NJ tree.

In [16]:
tree = build_and_plot_nj(adata, figsize=(10, 4), group_key="group", K_tree=2, show_leaf_labels=True, show_legend=True,
                         group_colors=group_colors, pc_loo=True, K_pc_loo=4, pc_loo_min_support=25)
